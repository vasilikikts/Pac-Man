# Pac-Man
Η εργασία υλοποιήθηκε στο πλαίσιο του μαθήματος Τεχνητή Νοημοσύνη του Τμήματος Πληροφορικής & Τηλεματικής (Χαροκόπειο Πανεπιστήμιο). Στόχος ήταν η εφαρμογή κλασικών αλγορίθμων αναζήτησης και τεχνικών πολυπρακτορικών συστημάτων στον κόσμο του Pacman.
Αρχεία που αναπτύχθηκαν
Η υλοποίηση έγινε με βάση το αρχικό project (project_1) και περιλαμβάνει τροποποίηση / συμπλήρωση των παρακάτω αρχείων:
  - search.py — Υλοποίηση αλγορίθμων αναζήτησης (DFS, BFS, UCS, A*)
   - searchAgents.py — Υλοποίηση πρακτόρων αναζήτησης και ευρετικών συναρτήσεων
  -  multiAgents.py — Υλοποίηση πρακτόρων MiniMax, Alpha-Beta και Expectimax
Κανένα από τα υπόλοιπα αρχεία του project (pacman.py, util.py, game.py κ.λπ.) δεν τροποποιήθηκε.
Περιγραφή της Ανάπτυξης
1) Αλγόριθμοι Αναζήτησης (search.py)
- Depth-First Search (DFS):
Υλοποιήθηκε αναζήτηση σε γράφο χρησιμοποιώντας στοίβα (Stack), με δομή κόμβου που περιλαμβάνει:
  - την τρέχουσα κατάσταση
  - τη μέχρι τώρα ακολουθία ενεργειών
  - τις προηγούμενες καταστάσεις για αποφυγή επαναληπτικών επισκέψεων
- Breadth-First Search (BFS)
Υλοποιήθηκε με χρήση ουράς (Queue), ώστε να επεξεργάζεται πρώτα τους ρηχότερους κόμβους.
Εξασφαλίζεται η εύρεση της συντομότερης διαδρομής σε περιβάλλον ίσου κόστους.
- Uniform Cost Search (UCS)
Χρησιμοποιήθηκε PriorityQueue, όπου προτεραιότητα αποτελεί το συνολικό κόστος.
Υποστηρίζονται μεταβαλλόμενα κόστη περιβάλλοντος.
- A* Search
Υλοποιήθηκε συνδυάζοντας:
   - κόστος διαδρομής
  - ευρετική εκτίμηση (π.χ. Manhattan distance)
Ακολουθείται πλήρως η λογική αναζήτησης σε γράφους με κλειστό σύνολο.
2) Πράκτορες Αναζήτησης & Ευρετικές Συναρτήσεις (searchAgents.py)
FoodSearchProblem + foodHeuristic
Υλοποιήθηκε ευρετική συνάρτηση για το πρόβλημα «Φάε όλες τις κουκίδες», συμβατή και αποδεκτή (consistent).
Η ευρετική σχεδιάστηκε ώστε:
  - να μειώνει δραστικά τον αριθμό επεκτάσεων σε σύγκριση με UCS
   - να παραμένει μη υπεραισιόδοξη (admissible)
   - να βελτιστοποιεί την ταχύτητα του A*
3) Multi-Agent Αναζήτηση (multiAgents.py)
- MinimaxAgent
Υλοποιήθηκε πλήρης πολυπρακτορικός αλγόριθμος:
  - με υποστήριξη πολλαπλών φαντασμάτων
  -  με υπολογισμό βάθους αναζήτησης
  -  με χρήση της έτοιμης evaluationFunction
- AlphaBetaAgent
Υλοποιήθηκε Minimax με κλάδεμα Alpha-Beta:
  - χωρίς κλάδεμα στην ισότητα (v > β / v < α)
     με διατήρηση της σειράς ενεργειών όπως ορίζονται από το GameState
- ExpectimaxAgent
Υλοποιήθηκε μοντελοποίηση μη-βέλτιστων αντιπάλων μέσω αναμενόμενης τιμής (expectation), υποθέτοντας ομοιόμορφη κατανομή των ενεργειών των φαντασμάτων.
4) Βελτιωμένη Συνάρτηση Αξιολόγησης
Στο betterEvaluationFunction εφαρμόστηκε συνδυαστική αξιολόγηση που λαμβάνει υπόψη:
  - απόσταση από φαγητό
- απόσταση από φαντάσματα (επικίνδυνα / φοβισμένα)
  - αριθμό κουκίδων που απομένουν
   - πιθανότητα παγίδευσης
   - βαθμολογία τρέχουσας κατάστασης (game score)
Στόχος ήταν υψηλή απόδοση σε παιχνίδι βάθους 2.
